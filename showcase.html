--text-base16/24 · 1.5
Body copy, the default. 1.5 line-height at this size also happens to be the WCAG 1.4.12 minimum for readable paragraph text — not a coincidence, a constraint that shaped where this step landed.

Seen in the wild — oblique

Code/oblique is the one project so far actually drawing on this system, though it doesn't <link> kit.css directly — it independently arrived at equivalent primitives under its own os.css. Side by side:

kit.css

.frame-thick {
  border: clamp(0.75rem, 4vw, 2rem)
    solid var(--color-frame);
}
.frame-thin {
  border: 2px solid var(--color-text);
}

oblique's os.css

.cool-border.thick {
  border: clamp(.75rem, 4vw, 2rem)
    solid var(--color-frame);
}
.cool-border.thin {
  border: 2px solid var(--color-text);
  /* + inset via top/right/bottom/left */
}

kit.css

.tag {
  border: 1px solid var(--color-accent);
  border-radius: 999px;
  padding: 0.2em 0.9em;
}

oblique's os.css

p.altview a {
  border: 1px solid var(--color-accent);
  border-radius: 999px;
  padding: .5rem 1.25rem;
  text-transform: uppercase;
  letter-spacing: .05em;
}

kit.css

--font-display: var(--font-sans);
/* a project earns its own by
   self-hosting an OFL typeface */

oblique's os.css

@font-face {
  font-family: 'Basteleur'; /* OFL-1.1 */
  src: url('fonts/Basteleur-Bold.woff2')
    format('woff2');
}
/* used for the strategy card only —
   Sligoil Micro (also OFL) everywhere else */

Not shown live on this page — the font files live in oblique's own repo, not here. The point isn't to reproduce oblique, it's that the same shapes (frame, pill, "earn your own display font") showed up independently, which is the evidence this kit is describing a real shared instinct and not inventing rules nobody was already following.

Contrast ratios computed via the WCAG relative-luminance formula against the exact rgb() values in kit.css — see NOTES.md changelog for when this page was generated.
